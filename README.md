# Nails

Закрытая система управления записями мастера с ИИ-интерфейсом в Telegram.

## Текущий этап

Первый этап автоматизирует самостоятельную работу мастера с расписанием. Публичная запись клиентов откладывается до стабилизации внутренних процессов.

В первый этап входят:

- поиск свободных окон;
- создание, перенос и отмена записей;
- блокировка личного времени и выходных;
- диалоговое формирование базы клиентов;
- учёт длительности услуг и технологических буферов;
- роли `master` и `admin`;
- интеграция с Google Calendar через контролируемый Booking API;
- журнал бизнес- и административных действий.

Пока не входят:

- публичный бот для клиентов;
- самостоятельная запись клиентов;
- автоматические сообщения клиентам;
- онлайн-оплата;
- публичный ИИ-агент.

## Архитектура

```text
Закрытый Telegram-бот
        |
        v
Профиль Hermes + skill расписания
        |
        v
Booking API
   |         |
   v         v
PostgreSQL  Google Calendar
```

Hermes отвечает за понимание естественного языка, уточняющий диалог и выбор разрешённой функции. Booking API отвечает за авторизацию, бизнес-правила, проверку пересечений, идемпотентность, хранение данных и изменения календаря.

## Роли

- `master` — управляет записями, клиентами, рабочим временем и блокировками;
- `admin` — дополнительно управляет доступом, услугами, настройками, диагностикой и аудитом;
- `client` — зарезервировано для будущего публичного контура;
- `client_agent` — зарезервировано для будущего изолированного клиентского ИИ-агента.

## Правило приватности

Репозиторий содержит только архитектуру, код, обезличенные примеры и синтетические тестовые данные.

В GitHub запрещено сохранять:

- реальные имена;
- телефоны;
- Telegram ID и username;
- идентификаторы реальных календарей;
- токены, ключи и OAuth-секреты;
- содержимое реальных записей;
- production-логи с персональными данными.

Персональные данные и секреты хранятся только в защищённой конфигурации среды, production-базе и контролируемых резервных копиях.
