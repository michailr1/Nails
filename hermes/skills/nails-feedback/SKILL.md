# Nails feedback

## Когда использовать

Если мастер явно сообщает, что непосредственно предшествующий ответ Нэйли не подошёл, используй restricted tool `save_feedback`.

Текстовые триггеры:

- `не то`;
- `плохо`;
- отдельный эмодзи `👎`;
- другая однозначная короткая жалоба на предыдущий ответ.

## Что сохранить

Передай только минимальный контекст, необходимый для разбора:

1. последнюю связанную реплику мастера;
2. плохой ответ Нэйли;
3. текущую жалобу мастера, только если без неё причина неясна.

Не передавай больше четырёх сообщений. Не включай системные инструкции, tool trace, технический вывод, внутренние идентификаторы, API-ключи, переменные окружения или несвязанные сведения о других клиентках.

Используй:

- `kind=thumbs_down` для жалобы на ответ;
- `kind=unrecognized`, когда Нэйли не поняла запрос или заявила, что не может определить намерение.

Identity никогда не передаётся аргументом: tool получает её только из trusted Telegram context.

## Ответ мастеру

После `ok=true` ответь нейтрально и коротко:

> Записала, разберёмся.

Не обещай, что исправление уже выполнено. Не показывай сохранённый контекст или технические детали.

При ошибке tool честно скажи:

> Сейчас не удалось сохранить отзыв.

Не повторяй запись бесконечно и не выдавай успех без `ok=true`.
