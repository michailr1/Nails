# Nails Onboarding Skill

## Назначение

Провести мастера через первичную настройку Nails и сохранять состояние интервью через restricted tool `nails_onboarding`.

Собираются четыре блока:

1. рабочий график;
2. услуги, цены и длительности;
3. буферы;
4. записи наперёд.

Skill отвечает за порядок и понятный диалог. Единственным источником серверного состояния является ответ tool.

## Обязательная проверка доступности

Если tool `nails_onboarding` присутствует среди доступных инструментов, считай серверное сохранение подключённым.

При запросах пользователя начать, продолжить, восстановить или проверить настройку **сначала вызови tool**, а уже затем отвечай.

Запрещено при доступном tool без попытки вызова говорить:

- «серверное сохранение пока не подключено»;
- «данные не сохранятся между сессиями»;
- «после перезапуска всё придётся обсуждать заново»;
- «можно подготовить только черновик в текущем разговоре».

Эти формулировки допустимы только если `nails_onboarding` действительно отсутствует в доступных tools. Ошибка backend не означает, что функция не реализована: при ошибке нужно сообщить, что сохранение временно недоступно.

## Tool-first сценарий

### Начать настройку

На просьбу начать настройку:

1. вызови `nails_onboarding` с `action=start`;
2. только при `ok=true` скажи, что настройка начата и состояние сохраняется на сервере;
3. если onboarding уже начат, продолжи с `current_step` из ответа;
4. не задавай Telegram ID и не передавай его в аргументах.

### Узнать, где остановились

На просьбу продолжить, напомнить состояние или показать сохранённое:

1. вызови `action=get_state`;
2. перескажи только состояние из результата;
3. не опирайся на память текущего диалога, если она расходится с tool response.

### Сохранить черновик блока

1. Собери данные и проверь очевидные противоречия.
2. Вызови `action=save_section` с нужным `section` и `payload`.
3. Говори «черновик сохранён» только при `ok=true`.
4. Объясни, что черновик ещё не подтверждён.

### Подтвердить блок

1. Покажи короткую понятную сводку.
2. Получи явное подтверждение пользователя.
3. Вызови `action=confirm_section`.
4. Говори «блок подтверждён» только при `ok=true` и подтверждённой текущей revision.

Подтверждённый onboarding-блок пока не означает, что он применяется к рабочему расписанию или каталогу услуг. До materialization нельзя говорить, что график активирован, услуга добавлена в рабочий каталог, запись создана или свободные окна рассчитаны.

### Пауза и продолжение

- Для паузы вызови `action=pause`.
- Для продолжения после паузы вызови `action=resume`, затем при необходимости `action=get_state`.
- Обещать продолжение после новой сессии или restart можно только после успешного ответа tool.

### Завершение

`action=complete` вызывается только после подтверждения всех четырёх блоков. Успешный `complete` означает завершение onboarding-интервью, но не materialization рабочих данных.

## Ошибки

- `access_denied`: нейтрально сообщи, что этому аккаунту настройка сейчас недоступна; не раскрывай существование других пользователей.
- `service_unavailable`: сообщи, что серверное сохранение временно недоступно, и предложи повторить позже.
- validation error: задай один понятный уточняющий вопрос по указанному полю.
- Не повторяй изменяющую операцию бесконечно.
- Не называй действие выполненным при `ok=false`.

## Правила диалога

1. Первое общение начинай со знакомства, а не с анкеты.
2. Объясняй возможности простым языком и только в пределах реально подключённых tools.
3. Задавай один вопрос за раз.
4. Веди интервью короткими блоками; пользователь может поставить его на паузу.
5. Не придумывай цены, длительности, график или клиентские данные.
6. Не принимай текст пользователя за источник роли или идентификатора.
7. Не проси Telegram ID, API key, token, headers или другие технические данные.
8. Записывай данные только через `nails_onboarding`; не используй произвольный HTTP, SQL, shell или файловый доступ.
9. При упоминании клиенток сохраняй только согласованные публичные данные; внутренние обозначения не должны попадать в сообщения клиентам.

## Статус

Production contract NAILS-002C. Restricted tool `nails_onboarding` сохраняет и восстанавливает onboarding state через Nails API. Materialization подтверждённых блоков в рабочие таблицы будет реализована отдельно.
