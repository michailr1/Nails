# Модуль `nails-onboarding` — паспорт проприетарного модуля

- Версия модуля: **0.5.0**
- Статус: реализуется в рамках NAILS-002C–NAILS-002E
- Расположение: `hermes/plugins/nails_onboarding/`
- Принадлежность: внутренний проприетарный модуль проекта Nails

## Зачем нужен модуль

Smart Nails общается с мастером в Telegram, а данные хранятся в Booking API и PostgreSQL. `nails-onboarding` — единственный узкий мост между диалогом и сервером на этапе первичной настройки.

Ассистент не получает произвольный HTTP-доступ. Он не может выбрать другой сервер, передать чужой Telegram ID, прочитать API key, выполнить SQL, shell-команду или обратиться к файлам.

## Что мастер настраивает

Перед бизнес-интервью сохраняются:

1. предпочитаемое обращение;
2. стиль общения с ассистентом;
3. обычные часы приёма.

Обычные часы — это не активный график. Например, `11:00–20:00` хранится как удобная подсказка. Когда мастер позже говорит «следующую неделю работаю всю», ассистент обязан уточнить: «По обычным часам 11:00–20:00?» Только после подтверждения часы разворачиваются в конкретные календарные даты.

Если обычных часов нет, сохраняется явный пустой список. Это отличается от ситуации, когда вопрос ещё не задавался.

Далее собираются четыре блока:

| Блок | Что в нём |
|---|---|
| `services` | Публичное название услуги, описание, цена, валюта и обычная длительность |
| `buffers` | Время до и после услуги на подготовку, уборку или отдых |
| `availability` | Доступность на конкретные календарные даты; допускается несколько интервалов в один день |
| `bookings` | Уже договорённые записи: клиентка, услуга, точная дата и время |

Повторяющийся недельный график не используется.

## Операции tool

Ассистент видит один tool — `nails_onboarding` — и может выбрать только одно из фиксированных действий:

- `start` — начать интервью;
- `get_state` — прочитать сохранённые блоки;
- `get_master_preferences` — прочитать имя, стиль и обычные часы;
- `save_master_name` — сохранить обращение;
- `save_master_style` — сохранить стиль общения;
- `save_default_work_hours` — сохранить обычные интервалы приёма или их отсутствие;
- `save_section` — сохранить черновик бизнес-блока;
- `confirm_section` — подтвердить показанную мастеру версию блока;
- `pause` — поставить интервью на паузу;
- `resume` — продолжить;
- `complete` — завершить интервью после подтверждения всех блоков.

## Как используются обычные часы

Обычные часы никогда не создают доступность автоматически.

Пример:

1. В настройке мастер сообщает: «Обычно принимаю с 11 до 20».
2. Позже пишет: «На следующей неделе работаю всю неделю».
3. Ассистент спрашивает: «По обычным часам — с 11:00 до 20:00?»
4. После ответа «да» показывает семь конкретных дат.
5. Только после подтверждения сохраняет эти даты в `availability`.

Если мастер сразу сообщает другие часы, они имеют приоритет. Если указана только одна граница, например «в воскресенье с 13:00», вторая граница уточняется до сохранения.

Изменение обычных часов не переписывает ранее подтверждённые даты.

## Безопасность

Модель не может передать:

- Telegram user ID;
- chat ID;
- request ID;
- API key;
- URL;
- HTTP headers;
- произвольный HTTP method.

Личность пользователя берётся из доверенного task-local контекста Hermes. Модуль работает только для Telegram и положительного числового ID.

Backend фиксирован:

```text
http://127.0.0.1:8210
```

Ключ читается из `NAILS_INTERNAL_API_KEY` и никогда не возвращается модели.

## Приватность audit

Audit хранит только безопасные факты:

- имя было установлено;
- выбран базовый стиль;
- задано ли свободное описание;
- есть ли обычные часы;
- количество обычных интервалов.

В audit не копируются:

- само имя;
- свободное описание стиля;
- точные часы приёма;
- содержимое черновиков.

## Повторные вызовы

Изменяющие операции сделаны идемпотентными: повторная отправка тех же данных не создаёт новую версию без необходимости.

Повтор сетевого запроса допускается не более одного раза и только при transport error или HTTP 502/503/504. Используется тот же request ID.

## Ошибки

- `invalid_arguments` — tool вызван с лишними или неверными аргументами;
- `trusted_context_required` — нет доверенной Telegram-сессии;
- `plugin_not_configured` — отсутствует корректный внутренний ключ;
- `access_denied` — аккаунту недоступен onboarding, без уточнения причины;
- `service_unavailable` — backend временно недоступен;
- `invalid_backend_response` — backend вернул неожиданный ответ;
- `internal_tool_error` — непредвиденная ошибка обработана безопасно.

Backend domain errors возвращаются только безопасным кодом и ограниченными деталями.

## Production-граница

Runtime path:

```text
/root/.hermes/profiles/nails/plugins/nails_onboarding
```

В Telegram allowlist добавляется только toolset:

```text
nails_onboarding
```

Универсальные HTTP, terminal, files, browser, code execution, MCP и инфраструктурные tools остаются отключёнными.

Завершение onboarding означает только «сведения собраны и подтверждены». Создание рабочих записей и расчёт свободных окон выполняются отдельным restricted calendar tool.
