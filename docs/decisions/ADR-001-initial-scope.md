# ADR-001: границы первого этапа

- Статус: принято
- Дата: 2026-07-12

## Контекст

Публичный клиентский бот требует отдельной модели угроз, изоляции, защиты от prompt injection, rate limiting и процесса подтверждения. Одновременно основная текущая проблема — ручное ведение расписания мастером.

## Решение

Первым реализуется закрытый агент для ролей `master` и `admin`.

Он автоматизирует:

- расписание;
- записи;
- переносы и отмены;
- блокировки времени;
- формирование клиентской базы в диалоге;
- каталог услуг и длительности;
- аудит действий.

Публичный контур клиентов переносится на следующий этап.

## Последствия

Преимущества:

- меньшая поверхность атаки;
- более быстрый пилот;
- возможность проверить реальные бизнес-правила;
- накопление качественной клиентской базы;
- подготовка стабильного Booking API для будущего публичного бота.

Ограничение:

- на первом этапе клиент не может самостоятельно записаться через бота.
