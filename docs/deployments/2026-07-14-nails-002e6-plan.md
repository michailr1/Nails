## Deployment actions

The runbook:

1. verifies root, hostname, paths, modes, branch and clean tree;
2. detects one approved baseline and its exact scheduling plugin version;
3. verifies the approved release contains PR #46;
4. rejects Alembic file changes and verifies revision `0006`;
5. verifies backend health/readiness, structured Hermes config and matching internal keys;
6. verifies current source/runtime equality before mutation;
7. creates a compressed PostgreSQL dump and root-only runtime backup;
8. fast-forwards to the exact approved release SHA;
9. builds the new API image while the old API remains online;
10. stops only the root user-level Nails gateway;
11. recreates only `nails-api` with `--no-deps`;
12. proves `nails-db` and Docker daemon remain unchanged;
13. installs scheduling plugin `0.3.0` and updated onboarding/scheduling skills;
14. proves Hermes config did not change;
15. verifies OpenAPI routes, plugin registry, Telegram visibility and the complete action set;
16. restarts only the root user-level gateway and scans its journal;
17. restores repository, API image/container, runtime/config/skills and gateway after any post-mutation failure.
