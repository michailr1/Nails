# WEB-STAT-001 — правило учёта прошедших записей

Ручное подтверждение результата визита не является обязательным условием для статистики.

- завершившаяся по времени запись `scheduled` по умолчанию считается состоявшимся визитом;
- её snapshot-сумма входит в общую выручку как предварительная;
- `completed` и подтверждённая итоговая цена входят в подтверждённую часть выручки;
- `no_show` и `cancelled` не входят в выручку;
- будущая `scheduled` запись не входит в заработок;
- `final_price_unknown` не превращается в ноль и учитывается отдельным счётчиком;
- ручная финализация остаётся способом уточнить исход и итоговую сумму, но не ежедневной обязанностью мастера.

API возвращает общую, подтверждённую и предварительную части отдельно, чтобы обычная статистика работала без ручного закрытия каждого визита.
